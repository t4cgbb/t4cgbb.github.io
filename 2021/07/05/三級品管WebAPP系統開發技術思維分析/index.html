一、技術

前端：HTML5, CSS3, Vue2-leaflet, leaflet.markercluster
後端：PHP
環境：Apache, MsSQL
版控：Git（初步導入使用）

備註：
同為土地系統開發案額外拓增的新需求項目，初期主要研究要思考 Native vs Hybird vs Web App 三者間的衡量，中後期則主要核心放在實踐新技術，看看是否能解決土地資產管理系統開發上的痛點（詳細可以去看那篇文章的說明）

二、開發功能特色介紹

流程圖點擊

實踐思路：功能十分簡單，不過因為是第一次實踐，所以特別寫下來，主要透過去實踐 Router 的跳轉。

群集功能

實踐思路：透過Leaflet.markercluster實踐群集功能，主要目的是為了避免打點過多造成移動時瀏覽器效能的下降。

電子簽名

實踐思路：透過 vueSignature 實踐電子簽名，再透過前端把簽名轉化成 Jpg 圖檔，轉成 Base64 編碼後發給後端 API。

獲取手機座標進行定位

實踐思路：透過 Leaflet 的 Locate 方法進行定位，若定位失敗會觸發 Locationerror，再針對失敗的狀態碼及訊息進行提醒及處理。

後端

  • 使用 Namespace，並遵循 PSR1, 2, 4 規範。

  • 使用 OOP 開發程式

    (使用 implements 去定義每個 Class 都要實作的 function，因為實踐邏輯一樣，僅參數不一樣，所以又定義了 Trait 的類別，直接在 Class 中 Use，大部分是這樣開發的)

  • 導入 Composer 並使用 Autoload（而不是大量的 Include）

  • 建立類似 MVC 的架構（統一入口，而不是各個請求對應一支 PHP）

  • 架構清楚整潔

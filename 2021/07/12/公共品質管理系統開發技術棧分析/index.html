一、技術

前端：Vue2, Vee-validate, vue-awesome, Vuex, ElementUI, Vue-router, Axios, Moment.js, Snow-calendar, V-charts
後端：Laravel5, Jwt-auth, Phpoffice, Tinybutstrong, Laravel-cors
環境：Apache, MsSQL
版控：Git
爬蟲：Python, Requests, BeautifulSoup, Pymssql, datetime, re

二、開發功能特色介紹

撰寫爬蟲，並透過排程方式定期撈取資料＆回寫資料

實踐思路：當初考慮 Selenium 及 Requests 實作，後來因為網站較單純就用 Requests 來實踐，透過選擇 tag 方式並用正則方式，將爬取到的資料進行清洗，再將清洗後的資料匯入資料庫。除此之外，當網頁資料更新時，也透過爬蟲方式將我方系統資料回寫到對方系統上，並透過 Windows.open 去彈出回寫後對方網站的訊息是否成功（因為對方全部都是 Html 呈現，沒有接口，故用該方法去實踐）

困難點：

  1. 網站時常更新，再加上網站的 Tag 基本沒有 id 之類可以去進行定位，會時常需要調整。
  2. 要爬的資料多，且規格有機會不一致，例如以為是數字突然變成中文，導致 Python 錯誤，後來再擷取資料過程中，陸續修正直至完善。
  3. 要注意同時間不要大量請求，以免被封鎖。

透過 JWT 實踐登入＆單一登入介接

實踐思路：案子過往使用 Session 來紀錄資訊，再後來同事使用自製 Token 來進行驗證，而再本系統我則透過實踐 JWT 登入，因為可以再裡面儲存一些可以公開的訊息，以便傳到前端時，可幫助前端作業，同時介接單位，單登的做法是由我們給予一個 API 進行申請，對方會將相關資訊傳至該 API，當該 API 取得相關資料時，與系統資料庫比對（例如用戶基本資料），若存在系統則登入，不存在則建立後登入。

行事曆功能

實踐思路：這邊透過台灣人撰寫的 Snow-calendar 進行實踐，但比較值得一提的是裡面有一些功能是沒有的，例如顏色的選擇等，故後來個人有拆解該原始碼去進行修正（詳細點我）

統計圖表實踐

實踐思路：透過 v-charts 實踐各種圖表以及下載，這邊僅部分，圖表類型至少十種以上（包含單純表格的統計亦有）

權限管理設計

實踐思路：系統從 ACL（Access Control List）一路進步到 RBAC（Role-Based Access Control）的設計，上圖便是單一角色的權限設計，透過帳號密碼傳至後端，並取得 Token，透過 Token 呼叫權限 API，會傳回權限配置，讓前端進行設定。

查核人員推薦功能


實踐思路：該部分是該系統最複雜的區塊，主要集中在後端，首先要透過名稱（這邊由客戶給予關鍵字）去進行專長匹配，並且匹配過程中有包含主要專長及次要專長，以及不能同時存在的人等，整個功能最後做出來會有點類似 AI（實際上不是 AI，也沒有自主學習的功能，因為還是程式去進行大量的判斷），然後進行人員的推薦。

查詢與即時運算

實踐思路：這部分列出來是因為如圖紅框，當設定很多條件後，系統會自動算出權重分數，原本是打算每次查詢時就進行即時的運算，後來因效能及頻率考量與規劃，改至會分成手動與排程兩種權重計算，並存在另外一張表再透過關聯的方式去產出查詢資料。

報名功能模組

實踐思路：如左下圖，透過 Vee-validate 實踐 Key in 時如不符合格式會跳出訊息提醒使用者，並且也同時在前端開發讓使用者透過上傳 Excel 報名資料，上傳後直接把報名人員填到報名單再讓使用者確認（不必一個一個報名填寫），算是一個全新的開發體驗，特別紀錄。

三、結論

本案幾乎結合了個人推廣到公司的所有技術，包含 Git、Vue 及 Laravel，同時結合了以前在家為了擷取評論而學習的爬蟲，對於框架的使用與模組的封裝（例如後端就封裝了 SqlConfing ＆ Api response）也更熟練，也對程式使用風格的規範，Php 為 Psr，Js 則使用 JavaScript Standard Style，爾後因應需求去當了 SA，並培養了 SA 的部分技能，詳細可查閱該篇文章。
